Git

    : 소프트웨어 버전관리 시스템(VCS) *참고 : SCM 개발자가 소스코드를 작성하고 관리하는 프로그램입니다. 개발과정에서 새로운 기능을 테스트하기 위해서 소스코드를 다양하게 구현하고 관리해야 합니다.

    - 협업에도 공유과 일관성 유지를 위해 필요합니다.

    - git으로 관리하면 변경된 이력이 저장됩니다.

    저장소 :

    일반적인 폴더와 유사한 개념이지만 관리하는 파일들의 변경된 이력을 저장하는 곳입니다.

    1. 로컬 저장소 : 내 PC에서 관리되는 개인 전용 저장소. local repository

    2. 원격 저장소 : 파일이 전용 서버에 관리되며 여러사람이 함께 공유하기 위한 저장소. remote repository

    3. push : 로컬 저장소에서 원격저장소로 파일 보내기

    4. pull : 원격 저장소에서 로컬저장소로 파일 가져오기

    로컬 저장소 만들기 : - 새롭게 직접 만들기 - 원격저장소에서 복사해오기 (clone,fork)

    커밋 commit : - 파일 및 폴더의 변경사항(추가/수정/삭제)을 저장소에 기록하기.(이력을 저장) - 버그 수정, 기능 추가 등 의미있는 변경사항을 구분해서 커밋. - 커밋 메시지 작성은 필수

    스테이징 staging : - 저장소에서 관리될 파일을 등록
